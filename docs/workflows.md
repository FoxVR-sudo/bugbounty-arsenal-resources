# Workflows

## Basic Recon Workflow

### Step 1 — Add Target
Provide a domain or URL for analysis.

---

### Step 2 — Run Recon
The platform performs:
- endpoint discovery
- header checks
- asset analysis
- subdomain enumeration

---

### Step 3 — Review Results
Analyze:
- discovered endpoints
- response codes
- JavaScript references
- headers

---

### Step 4 — Export Findings
Export findings for further manual validation or reporting.

---

## Recommended Targets

Use intentionally vulnerable applications or authorized assets only.

Examples:
- testphp.vulnweb.com
- demo.testfire.net
